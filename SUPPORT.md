# Support

We are dedicated to building a strong community and providing support for PAWPHILE users and contributors. Since PAWPHILE is an open-source project, our support channels are community-driven.

## Where to get help

### 1. Documentation
Before asking a question, please ensure you have read through our comprehensive documentation:
- [README.md](README.md) - For a high-level overview, architecture, and feature set.
- [docs/architecture.md](docs/architecture.md) - For detailed system design.
- [docs/setup.md](docs/setup.md) - For local environment configuration.
- [docs/api.md](docs/api.md) - For interacting with the backend and Vision services.

### 2. GitHub Issues
If you have found a bug or have a feature request, please open a [GitHub Issue](https://github.com/ESSAKKI-RAJA/PAWPHILE/issues).
- **Bug Reports**: Please use the Bug Report template and provide as much detail as possible (OS, Node version, Python version, steps to reproduce, and logs).
- **Feature Requests**: Please use the Feature Request template to propose new functionality.

### 3. GitHub Discussions
For general questions, architectural discussions, or community help (e.g., "How do I deploy this to AWS?", "What is the best way to fine-tune the Vision AI model?"), please use [GitHub Discussions](https://github.com/ESSAKKI-RAJA/PAWPHILE/discussions).

## Security Support
If you have discovered a security vulnerability, **do not open a public issue**. Please refer to our [Security Policy](SECURITY.md) and email the maintainers directly.

## Commercial Support
Currently, PAWPHILE is a research-oriented, open-source project. If you are an enterprise, veterinary clinic, or research institution looking for dedicated SLAs, custom integrations, or managed hosting, please contact the maintainers directly via email.
